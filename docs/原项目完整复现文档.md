## 1. 项目概述

基于Barra多因子模型的风格因子择时策略系统。核心流程：

1. **因子计算**：从MySQL提取A股行情+财务数据，计算25个(CNE6)或10个(CNE5)风格因子
2. **因子收益**：横截面OLS回归计算因子日收益率及累计收益率
3. **择时信号**：通道突破策略(Donchian Channel)，生成信号(+1/0/-1) → 最优向量(+3/0/-3)
4. **股票选取**：余弦相似度选股，选相似度最高的top_n只
5. **周度换仓**：等权重持有，记录交易

- **CNE6**（主用）：25因子，top_n=100
- **CNE5**（备选）：10因子，top_n=50
- 回测期：2020-01-01 ~ 2025-12-31，数据回溯400天

#### 因子计算公式

| 因子类别 | 因子 | 公式 | 数据源 |
| --------- | ------ | ------ | -------- |
| 规模 | SIZE | log(total_cap*10000) | 行情 |
| 规模 | MIDCAP | SIZE^3 对SIZE截面回归取残差 | 行情 |
| 波动 | BETA | rolling(252).cov(ret,mkt_ret)/rolling(252).var(mkt_ret) | 行情 |
| 波动 | RESVOL | rolling(252).std(ret - BETA*mkt_ret)*sqrt(252) | 行情 |
| 波动 | HISTVOL | rolling(252).std(pct_chg)*sqrt(252) | 行情 |
| 动量 | MOMENTUM | (close_21d/close_252d-1) - (close/close_21d-1) | 行情 |
| 动量 | RESMOM | rolling(252).mean(ret-BETA*mkt_ret)*252 | 行情 |
| 价值 | BTOP | TotalShareholderEquity/(total_cap*10000) | BS |
| 价值 | EYIELD | NPParentCompanyOwners/(total_cap*10000) | IS |
| 价值 | CFP | NetOperateCashFlow/(total_cap*10000) | CF |
| 价值 | SP | OperatingRevenue/(total_cap*10000) | IS |
| 价值 | LP | TotalLiability/(total_cap*10000) | BS |
| 成长 | EGRO | NPParentCompanyOwners / prev(NPParentCompanyOwners, shift4) -1 | IS |
| 成长 | SGRO | OperatingRevenue / prev(OperatingRevenue, shift4) -1 | IS |
| 杠杆 | MLEV | mkt_cap/(mkt_cap-TotalLiability) | BS |
| 杠杆 | BLEV | TotalAssets/TotalShareholderEquity | BS |
| 流动 | LIQUIDITY | log(turnover+1) | 行情 |
| 流动 | STOM | rolling(21).mean(turnover) | 行情 |
| 流动 | STOQ | rolling(63).mean(turnover) | 行情 |
| 盈利 | ROE | NPParentCompanyOwners/TotalShareholderEquity | IS+BS |
| 盈利 | ROA | NPParentCompanyOwners/TotalAssets | IS+BS |
| 投资 | CAPX | FixIntanOtherAssetAcquiCash/TotalAssets | BS+CF |
| 投资 | AGRO | TotalAssets/prev(TotalAssets, shift4)-1 | BS |
| 其他 | TOPSI | rolling(252).sum(pct_chg) | 行情 |
| 其他 | SEASON | (cumsum(pct_chg,by code+month) - pct_chg) / cumcount(,by code+month) | 行情 |

## 7. 数据偷看（Look-ahead Bias）风险与防护

### 7.1 财务数据使用ann_date而非report_date

**风险**: report_date是报告期(如2024Q1=2024-03-31)，但实际公告可能在4月底。如果按report_date匹配，会提前使用未公告数据。

**防护**: 所有财务因子计算均通过`_fast_financial_merge()`使用`pd.merge_asof(direction='backward')`按ann_date匹配，确保只使用已公告数据。

```python
# 错误做法（会偷看）:
merged = pd.merge(price_df, financial_df, on=['date','code'])  # 用report_date

# 正确做法:
merged = pd.merge_asof(price_sub, financial_df,
    left_on='date', right_on='ann_date',
    by='code', direction='backward')  # 只用已公告数据
```

### 7.2 标准化顺序：先过滤再标准化

**风险**: 如果先标准化全历史数据（包括回溯期），标准化统计量(均值/标准差)会包含未来信息。

**防护**: `run()`中先过滤到`start_date`，再标准化。

```python
# 错误做法:
factor_exposure_df = standardize(factor_exposure_df)  # 用全历史统计量
factor_exposure_df = factor_exposure_df[date >= start_date]

# 正确做法:
factor_exposure_df = factor_exposure_df[date >= start_date]  # 先过滤
factor_exposure_df = standardize(factor_exposure_df)  # 再标准化
```

### 7.3 信号日期：使用前一日因子暴露

**风险**: 在current_date选股时使用current_date的因子暴露，但因子暴露依赖当天收盘价，实际不可获得。

**防护**: `signal_date = all_dates[current_idx - 1]`，用前一个交易日的因子暴露。

```python
current_idx = all_dates.index(current_date)
signal_date = all_dates[current_idx - 1] if current_idx > 0 else current_date
```

### 7.4 择时信号：shift(short_prd)避免噪音和未来

**风险**: 通道突破直接用当日值判断，会因最新几天噪音产生虚假信号。

**防护**: `shift(5)`偏移5天，用5天前的累计收益率计算通道。

```python
bf_max = cumulative_returns[factor].shift(5).rolling(20).max()
bf_min = cumulative_returns[factor].shift(5).rolling(20).min()
```

### 7.5 SEASON因子：排除当前月

**风险**: 季节性因子如果包含当月数据，等于用当天收益预测当天收益。

**防护**: `(cumsum - current_pct) / count`，分子减去当前月收益。

```python
pct_cumsum = groupby(['code','month'])['pct_chg'].cumsum()
pct_count = groupby(['code','month']).cumcount()
SEASON = (pct_cumsum - pct_chg) / pct_count  # 不含当前月
```

### 7.6 TOPSI因子：rolling(252)只看过去

**风险**: 如果使用expanding窗口，会包含从上市以来的全部历史，可能隐含偷看。

**防护**: 固定252天滚动窗口。

```python
TOPSI = groupby('code')['pct_chg'].transform(lambda x: x.rolling(252, min_periods=252).sum())
```

### 7.7 因子收益率回归中的shift(-1)

**说明**: `ret = pct_chg.shift(-1)` 看似偷看，但这是正确的：

- 在日期T，因子暴露已知，回归目标是T+1的收益率
- 这是因子收益率的定义（用今日因子解释明日收益），不是策略偷看
- 策略选股用的是signal_date（昨天）的因子暴露

## 10. ST/停牌股票过滤机制

### ST股票 — SQL级别永久排除

```sql
AND is_st = 0  -- 在get_stock_data()中
```

**理由**: ST是相对永久的属性，从数据源头排除避免干扰因子计算。

### 停牌股票 — 策略级别动态排除

```python
# 在run_weekly_rebalance()中构建查找表
suspend_lookup = price_df[is_suspend==1].groupby('date')['code'].apply(set).to_dict()

# 在select_stocks()中过滤
for idx, row in current_data.iterrows():
    if row['code'] in suspended_codes:  # 跳过停牌
        continue
    # ... 计算相似度
```

**理由**: 停牌是每日动态状态，不影响因子暴露的时间序列完整性。停牌股票仍参与因子计算（保留历史连续性），但不在换仓日被选入组合。
